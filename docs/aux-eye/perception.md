# 辅眼感知规格

> **读者：AI 代理（执行辅眼观测流程的 agent）**
> 本文档描述辅眼是什么、它观察什么、以及观测数据如何产出与校验。这是一份规格说明，不是发给某个模型的系统提示词。

---

## 1. 辅眼定位

主眼是串口遥测：设备通过固件上报的数值序列（位置、速度、姿态角、状态字）。这类数据精确、高频，但只能反映设备"自己算出来"的内部状态——传感器融合的结果，不是物理世界的直接画面。

辅眼补的是主眼**测不到**的那一层：

- **绝对姿态相对重力/地面的关系**——遥测里的姿态角来自内部传感器积分，可能存在漂移或标定误差；辅眼从画面里看设备相对地面/水平线的真实朝向。
- **空间位置漂移**——设备在地面上的绝对位置和移动轨迹，遥测通常不直接给出（除非有外部定位），辅眼靠画面观察设备是否偏离了预期区域。
- **部件级几何状态**——某个可动部件是否达到了预期的角度/位置/形态，这是几何观察，遥测的数值不一定能覆盖每个部件。
- **失败形态**——设备倾倒、脱离画面、被遮挡等失败状态，往往在遥测数值上表现得不明显（甚至遥测本身还在正常上报），但画面上一目了然。
- **可见振荡**——肉眼能看到的抖动、摆动、周期性晃动，即便遥测采样率不足以捕捉完整波形，画面也能给出直观判断。

这些能力是**通用的场景观察能力**，不绑定任何具体设备构型的判断逻辑——辅眼看到的是"姿态、位置、几何、失败、振荡"这类通用概念，具体设备是什么由观测时的场景决定，不是辅眼本身内置的知识。

## 2. 数据流

辅眼观测走一条固定的四段流水线：`tools/camera_capture.py` 打开摄像头抓一帧（或多帧），把画面存到磁盘并把每帧的元信息（路径、sha256、序号、时间戳、宽高）以 NDJSON 形式打到 stdout；执行 agent 用 `look_at` 读取该帧文件；agent 依据这一帧产出一份结构化的观测 JSON；最后用 `tools/aux-eye/verify_aux_eye.py`（校验器，接收帧路径与观测 JSON）做 schema 校验与帧身份比对，确认这份观测确实对应那一帧、字段形状合法。整条链路里，画面理解发生在 agent 读帧这一步，`camera_capture.py` 本身只做纯采集，不做任何画面理解或网络调用；校验发生在链路末端，只判定结构与身份，不评判描述措辞。

## 3. 观测 JSON

观测 JSON 的字段定义、类型约束与合法性规则，唯一真相源是 `schemas/aux-eye/perception.schema.json`（JSON Schema draft-07）。产出或校验观测 JSON 时应直接引用这份 schema 文件，不要在别处复制字段定义——本节只解释字段的语义，不重复 schema 里的类型细节。

必填字段是三个"状态/交接"字段，它们与画面理解能力无关，始终应当能产出：

- `source_frame_sha256` —— 该帧文件的 SHA-256 十六进制摘要。这个值由抓帧/校验流程计算，不是由读图这一步"看出来"的，作用是加密级证明这份观测确实对应磁盘上那一帧，防止张冠李戴。
- `visible` —— 这一帧是否读到了可用内容的布尔状态信号。它的存在是为了给出一个保守、fail-safe 的判断：读不到就应该是 `false`，不应该在不确定时声称"看见了"。
- `failure_reason` —— 当读不到内容时说明原因的枚举值（暗、遮挡、空、或"无失败"）；`visible` 与 `failure_reason` 之间有一致性约束，具体逻辑见 schema 文件里的 `if`/`then`/`else`。

可选字段是依赖真实画面理解能力的"内容"字段：`scene.subject`（场景主体的简短描述）、`objects`（画面中识别出的物体列表，每项带名称和自评置信度）、以及顶层 `confidence`（整体自评置信度）。这些字段的取值质量取决于执行该观测的模型是否具备真实的图像输入能力。根据 `.omo/evidence/vision-probe.md` 的实测：在不具备图像输入能力的执行会话里，这些内容字段会自然产出为空字符串、空数组或缺省，而不是被强行编造出内容；schema 把它们设为可选正是为了让这类观测依然是一份合法的观测。在具备真实视觉能力的执行会话下，这些字段应当被正常填充。换句话说，这份规格对两种会话都成立：内容字段"能填就填，填不了就留空"，永远不需要为了凑格式去编造。

一个通过 schema 校验的示例观测：

```json
{
  "source_frame_sha256": "a2549799d39a127b493a7638f06e619e6c1b688af880cfd37de2752022c11548",
  "visible": true,
  "failure_reason": "none",
  "scene": {
    "subject": "a mug on a wooden desk"
  },
  "objects": [
    {
      "name": "mug",
      "confidence": 0.82
    }
  ],
  "confidence": 0.75
}
```

## 4. 可选 ArUco 增强

当画面中存在一个 ArUco 标记时，`tools/aruco_pose.py` 可以在辅眼的默认观测之外，叠加一份基于标记几何解出的定量偏航角/位姿数据，把"大致朝哪个方向"变成一个可复现的角度数值；当画面中没有标记时，默认的观测路径完全不受影响，照常产出。这是一个可选增强模块，不是辅眼观测流程的必需环节。

---

## 5. 应用示例：嵌入式/调试时可关注项

- 运动机械部件的姿态是否符合预期
- 屏幕/指示灯上的文字或状态
- 画面中是否有意外遮挡或位置漂移（例如平衡车这类设备偏离了预期区域）
