# drivers/ — 自研可移植设备驱动

## 这一层放什么

**自研的、可跨芯片/跨板复用的外部设备驱动**：传感器、显示屏、Flash、按键矩阵、电机驱动等。

（本任务先建空占位，后续按需填充。）

典型组织：

```
drivers/
├── sensor/
│   └── <device>/        # 如 bmp280、mpu6050
├── display/
│   └── <device>/        # 如 ssd1306、st7789
└── ...
```

## 什么该放这里

- 面向具体外设、但**不绑定单一芯片/板子**的驱动实现。
- 通过抽象接口（GPIO/I2C/SPI 句柄由上层注入）与硬件解耦，保证可移植。

## 什么不该放这里

- 芯片厂商的外设库/HAL（属于 `bsp/`）。
- 引脚映射与时钟（属于 `boards/`）。
- 通用中间件如 RTOS、文件系统内核（属于 `middleware/`）。
- 产品业务逻辑（属于 `apps/`）。

## 命名约定

- 按外设类别分子目录，再按型号分：`drivers/<category>/<device>/`。

## 与相邻层的关系（依赖方向）

驱动位于 middleware 同级、应用之下。它面向 `bsp/` 或板级提供的总线接口编程，但**不直接写死引脚**——引脚由 `boards/` 或 `apps/` 通过参数/句柄注入。
被 `apps/` 按需链接使用。依赖方向：`app → driver`（driver 不反向依赖 app）。
